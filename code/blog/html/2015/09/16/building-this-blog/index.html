Building a Github hosted Blog.

Sep 16, 2015

For the first code post on my shiny new blog, what is more appropriate then talking about the tech behind this blog? I wanted something hackable and flexible, but 100% clean and maintainable.

I haven’t yet posted the build source to github, but plan to soon. If anybody wants it now, then feel free to fire me an email.

Setting up hosting on github.

I chose github pages for hosting because its source controlled, free, and open. Here’s a basic walkthrough of how I setup the hosting.

  1. Find and purchase a domain name : Namecheap is a fairly easy and affordable option that gives you pretty solid management tools.
  2. Set up a CNAME record : Open your DNS management tools. For your domain there should be a USERNAME.github.io. in the alias field, enter and you should be good to go. This tells all visitor’s browsers that the domain name is served by github’s domain name (canonical name) even though to the user it appears as your URL.
  3. Create a github repository : for your site.
  4. ** Create a file ** : Name the file./CNAME in the root of the repo. This tells github that you’re redirecting a url to them. You can verify that it worked by checking the Github pages section in the repo’s settings.
  5. **touch index.html **: file in the root of your repository.
  6. Check your new URL in the browser and make sure its working. It might take a little bit for the DNS records to update.

Setting up a deployable repo.

This is the biggest long term decision for getting your site running. Once you have it up and running, your probably not going to want to mix it up. There are tons of options for static sites– makes things easy, right?

Now there are plenty of options for frameworks, so I’m going to leave that decision up for you. Here’s a nice compilation showing static site gens by popularity on github.

Here are some factors to consider:

  1. Whats the build process? I prefer grunt but there are other options. Some are standalone processes in their own langauges, some use grunt/gulp for node.js, and some use combinations of both.
  2. Whats the templating engine? Ultimately this comes down to personal preference, so choose one that you’re familiar with or works well with your language/toolset choices. I’m a big fan of using markdown for writing blogposts and plenty of frameworks support that.

One Command Deploys

All of my tooling builds into one directory which can be commited directly to the branch for your gh-pages site. I use grunt’s gh-pages toolset. You also need to have a backup of your build process somewhere. You can make a new repo or commit it to a branch on the same repo.
